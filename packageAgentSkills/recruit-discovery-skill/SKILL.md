# 公开招募检索

检索某场电音节的公开组队招募帖并展示列表卡片；仅信息检索，不配对撮合、不生成草稿、不发布、不保证组满。

## 触发场景

用户原话举例（路由命中本技能）：
- 「帮我找去 EDC Korea 的公开组队帖」
- 「上海出发、3 人左右的 techno 招募」
- 「这场节有哪些人在找队」
- 「泰国 TML 组队」
- 「看看有没有一起去的」

## 不适用范围

- 智能匹配搭子、平台担保组满、私信配对 → 合规红线，不在本技能范围
- 自动发帖、代发招募、生成招募草稿 → 微信 AI 不处理，请用户进入小程序手动操作
- 收藏、关注、报名、私信、订阅提醒 → 微信 AI 不处理

## 前置条件

- 须已知具体活动（用户已选定或上文检索到节名）
- 公开招募列表无需登录；本技能不触发登录，不写入任何用户数据

## 使用顺序

**一步直达**：`searchPublicRecruits` 已内置活动查找——直接传入用户完整意图（如 `{ query: "泰国tml 组队" }`），后端自动完成「活动匹配 → 帖子检索」。**无需**先调 `searchFestivals`。

调用后**必须立刻渲染 `recruit-list-card`**（无论结果 0 条或多条）。

## 强制规则（MUST）

1. **卡片渲染强制规则**：调用 `searchPublicRecruits` 后**必须立刻渲染 `recruit-list-card`**。无论是 0 条、1 条还是多条，卡片都必须出现。**禁止**仅有文字回复。API 返回的 `uiDirectives` 里指定了必须渲染的卡片组件。

2. **活动名称强制规则**：文字回复和卡片内容**必须使用 `searchPublicRecruits` 返回的 `activityName` 完整官方名称**，严禁使用用户输入的简称。

3. **筛选条件传递规则**：用户提出的人数、出发地、曲风等筛选条件**必须传入 `searchPublicRecruits` 的 `query` 参数**，不得仅靠文字描述来"筛选"。卡片需展示对应的 filterLabels。
   - 用户要求「4 人没有再查 3 人」时，必须先用包含「4人」的 query 查一次并渲染卡片；若 0 条，再用包含「3人」的 query 查第二次并渲染第二张卡片。

4. **条件分支执行规则**：用户要求「先查 A 活动，没有则查 B 活动」时：
   - Step 1：`searchPublicRecruits({ query: "A活动 筛选条件" })` → **渲染 `recruit-list-card`**
   - Step 2：仅在上一步返回 `activityMismatch` 或 0 条时，调用 `searchPublicRecruits({ query: "B活动 筛选条件" })` → 渲染卡片
   - **禁止**不渲染卡片；跳过 A 直接查 B

5. **只读边界强制规则**：本 skill 只能调用 `searchPublicRecruits`（以及必要的活动检索上下文能力）。用户要求写招募、生成草稿、发布、报名、收藏、关注、私信时，**禁止**调用或暗示任何写入能力；只能说明「微信 AI 当前只提供公开招募查询，请进入小程序手动操作」。

## 回复与展示要求

- 检索完成后**须展示招募列表卡片**（`recruit-list-card`），无论结果数量多少
- 有结果时：文字中概括首条招募帖要点，并说明还有多少条；卡片展示筛选标签
- 无结果时：明确告知「暂无公开招募帖」，卡片展示空状态，不引导起草或发布
- 卡片点击打开活动详情招募墙（半屏）
- 活动名称使用 `searchPublicRecruits` 返回的 `activityName`
