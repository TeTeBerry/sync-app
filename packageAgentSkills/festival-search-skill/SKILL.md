# 电音节资讯检索

基于活动 catalog 检索电音节、查看单场详情与官宣阵容；仅提供公开资讯，不售票、不撮合组队。

## 触发场景

用户原话举例（路由命中本技能）：
- 「EDC Korea 什么时候」
- 「查一下泰国 TML」
- 「这场阵容出了吗」
- 「最近有什么电音节」
- 「Tomorrowland 几号」
- 「帮我检索所有电音节活动」

## 不适用范围

- 找队友、智能配对、平台担保组满 → 由招募相关技能或小程序内操作处理
- 发布招募帖、代发帖 → 须进入小程序确认，不在本技能内完成

## 前置条件

- 检索与查看阵容为公开能力，无需登录
- 订阅阵容更新、生成招募草稿等写操作需登录（其它技能）

## 使用顺序

- 查看阵容前需先通过检索或详情拿到具体活动
- 用户只问节名时先检索活动列表，再视需要拉详情或阵容

## 强制规则（MUST）

1. **活动名称强制规则**：所有文字回复、卡片展示、工具调用结果描述中，**必须使用 `searchFestivals` 或 `getEvent` 返回的 `events[].name` 字段值**（即 catalog 完整官方活动名称），**严禁**使用用户输入的简称、缩写或口语化名称。即使在多轮对话上下文中已提及该活动，每次指代仍须使用完整官方名称。
   - ✅ 正确：「Tomorrowland Thailand 2026」
   - ❌ 错误：「TML 泰国」「泰国的 Tomorrowland」「Tomorrowland 泰国 2026」
   - **无匹配活动时**：明确告知用户「catalog 中暂无该活动的完整官方名称」，**禁止**直接沿用用户输入的名称。

2. **卡片渲染强制规则**：调用 `searchFestivals`、`getEvent`、`getLineup` 后**必须**让对应的 UI 卡片渲染指令生效（`search-results-card`、`event-card`、`artist-lineup-strip`），**禁止**仅有文字回复而无卡片。卡片是此技能的唯一视觉契约。

3. **曲风筛选规则**：用户要求按曲风筛选时，先调用 `searchFestivals` 检索活动，再对返回结果按活动名称中的曲风特征（如 Hardstyle、Techno、House）做匹配过滤；若 `searchFestivals` 的 query 不命中，须将曲风作为独立的过滤条件在返回结果上执行，**禁止**仅将曲风作为 query 参数传入就结束。

## 回复与展示要求

- 用户提到出发城市时，先记录检索偏好（不申请定位权限），再检索活动
- 检索**仅 1 场**时展示单场资讯卡（标题为活动全名）；**多场**时展示横滑「电音节对比」卡片
- 调用详情后，文字回复与活动卡片须包含**举办时间、地点**等基础字段；若用户只问基础信息，优先展示这些字段再视需要查阵容
- 阵容结果展示**横滑艺人条**，保留活动名称、时间、地点
- 用户要求筛选、排序或对比多场活动时，须**列出符合条件的完整活动清单**，并**说明排序或筛选规则**（如按艺人数量降序），再指出首选场次
- 活动卡片支持分享与查看场地（有坐标时）
- 检索结果为 0 时：先告知无匹配结果，再引导用户尝试更具体的活动名称或查看全量活动列表
