# 观演准备

订阅活动阵容更新提醒，或创建出行攻略生成任务；内容仅供参考，不保证票务与组满。

## 触发场景

用户原话举例（路由命中本技能）：
- 「订阅阵容更新」「阵容出了提醒我」
- 「生成出行攻略」「从上海怎么去」
- 「帮我做个观演准备」
- 「这场节攻略怎么弄」

## 不适用范围

- 查节、查招募、写招募帖 → 由检索/招募类技能处理
- 购票、抢票、保证成行 → 不在本技能范围

## 前置条件

- 订阅阵容与生成攻略均需登录
- 生成攻略须已知活动、出发地、人数等信息

## 使用顺序

- 须先确定具体活动
- 订阅阵容与生成攻略可独立进行，无强制先后

## 强制规则（MUST）

1. **订阅前告知规则**：调用 `subscribeLineupUpdates` **之前**，**必须**先向用户明确说明以下两点（不可略过直接调用接口）：
   - 阵容官宣或变更时将通过**微信订阅消息**推送提醒
   - 需要用户**授权微信订阅消息模板**
   调用后，再告知用户当前环境可能无法自动弹窗，引导用户点击卡片进入活动详情页亲手完成授权。

2. **攻略前确认规则**：调用 `generateTravelGuide` **之前**，**必须**先向用户复述并确认：出发城市（如广州）和人数（如 2 人）。即使用户消息中已包含这些信息，仍需确认后再调用。**禁止**未经确认直接调用接口。格式示例：「确认一下：从广州出发、2人同行，我帮你生成出行攻略？」

3. **攻略缺项追问规则**：用户未说明出发地或人数时，须先追问（如「从哪个城市出发、几人？」），**禁止**猜测或省略出发地直接调用 `generateTravelGuide`；检索时记录的出发城市仅可作追问参考（「是否仍从上海出发？」），不能代替用户确认。

4. **活动名称强制规则**：所有文字回复、卡片展示、工具调用结果描述中，**必须使用 catalog 返回的完整官方活动名称**，**严禁**使用用户输入的简称或自行缩写。订阅、攻略等写操作前须先确认活动完整名称。活动不存在时须明确告知，**禁止**使用用户输入的简称指代。

## 回复与展示要求

- 订阅阵容更新前，须说明：**阵容官宣或变更时将通过微信订阅消息推送提醒**（需用户授权订阅消息模板）
- 订阅流程会自动尝试弹出微信授权；**若 Agent 代调接口无法弹窗**（无用户点击手势），卡片会提示「待微信授权」，用户**点击卡片进入活动详情**后亲手点「订阅」完成授权
- 订阅成功时确认活动完整名称，并提示用户留意微信服务通知
- 攻略任务创建后引导用户点击卡片进入半屏攻略页查看进度；创建前须确认**出发地**与**人数**，缺一项则先追问用户
